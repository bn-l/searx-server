- 2026-03-25 20:46: Fixed Google engine cold-start suspension problem. After the three-repo split, Google searches weren't appearing — proxy fleet cold-starts (browser init + CAPTCHA solving) exceeded SearXNG's default 2s `outgoing.request_timeout`, causing `SearxEngineAccessDeniedException` which suspended Google for 180s with exponential backoff. Four changes across both repos: (1) `settings.yml` — raised `request_timeout` to 10s, `max_request_timeout` to 90s, Google engine `timeout` to 60s, set `suspended_times` to 0 for `SearxEngineAccessDenied` and `SearxEngineCaptcha` (disable suspension entirely — proxy fleet handles its own retries). (2) `proxy_service.py` — added FastAPI lifespan handler to warm up the browser at startup instead of lazy init on first request; `/status` now returns 503 when browser is `None` so Caddy treats cold instances as unhealthy. (3) `lb/Caddyfile` — added `lb_try_duration 30s` + `lb_try_interval 1s` for automatic failover to next proxy instance on failure (matching what `caddy-server` already does for SearXNG fleet); reduced `health_interval` to 5s, `health_fails` and `max_fails` to 1 for faster detection. (4) Created SearXNG plugin `engine_monitor.py` — hooks into `on_result`/`post_search` to track which engines return results; logs warning when Google is missing, escalates to error after 5 consecutive misses; guards against false positives by only monitoring when ≥2 other general engines returned results (filters out bang/category searches). Plugin mounted into all 10 containers via docker-compose volume and registered in `settings.yml`.
